代码使用提醒

本页代码用于学习和研究。复制到正式 WooCommerce 网站前，请先备份，并优先在测试环境验证。

凡是影响价格、库存、支付、订单、邮件和客户数据的代码，都需要完整测试购物车、结账、付款、退款、邮件和后台订单处理流程。

1. 读取订单基础数据

基础 · php
<?php $order = wc_get_order( 123 ); if ( $order ) { $email = $order->get_billing_email(); $total = $order->get_total(); $status = $order->get_status(); echo esc_html( $email . ' / ' . $status . ' / ' . $total ); }
订单读取优先使用 wc_get_order。

2. 更新订单 meta 并保存

基础 · php
<?php $order = wc_get_order( 123 ); if ( $order ) { $order->update_meta_data( '_sales_owner', 'Felix' ); $order->save(); }
HPOS 兼容代码应使用订单对象 CRUD。

3. 添加订单备注

基础 · php
<?php $order = wc_get_order( 123 ); if ( $order ) { $order->add_order_note( 'Customer requested installation guide.' ); }
订单备注适合记录内部跟进或自动化处理结果。

4. 查询最近处理中订单

实用 · php
<?php $orders = wc_get_orders( array( 'limit' => 20, 'status' => array( 'processing' ), 'orderby'=> 'date', 'order' => 'DESC', ) ); foreach ( $orders as $order ) { echo esc_html( $order->get_order_number() ); }
wc_get_orders 是 HPOS 友好的订单查询方式。

5. 监听订单状态变化

重要 · php
<?php function mywoo_order_status_changed( $order_id, $old_status, $new_status, $order ) { if ( 'processing' === $new_status ) { $order->add_order_note( 'Order moved to processing. Start fulfillment workflow.' ); } } add_action( 'woocommerce_order_status_changed', 'mywoo_order_status_changed', 10, 4 );
订单状态变更是自动化流程的关键 hook。

6. 检测 HPOS 是否启用

进阶 · php
<?php use Automattic\WooCommerce\Utilities\OrderUtil; if ( class_exists( OrderUtil::class ) && OrderUtil::custom_orders_table_usage_is_enabled() ) { error_log( 'HPOS custom order tables are enabled.' ); } else { error_log( 'Orders may be using legacy post storage.' ); }
一般业务代码优先用 CRUD，只有写底层 SQL 时才需要判断 HPOS。

本页总结

订单开发必须以 CRUD 为核心，尤其在 HPOS 环境下不要直接依赖订单存储在 wp_posts/wp_postmeta。
