代码使用提醒

本页代码用于学习和研究。复制到正式 WooCommerce 网站前，请先备份，并优先在测试环境验证。

凡是影响价格、库存、支付、订单、邮件和客户数据的代码，都需要完整测试购物车、结账、付款、退款、邮件和后台订单处理流程。

1. 遍历订单商品

基础 · php
<?php $order = wc_get_order( 123 ); foreach ( $order->get_items() as $item_id => $item ) { $product = $item->get_product(); $name = $item->get_name(); $qty = $item->get_quantity(); echo esc_html( $name . ' x ' . $qty ); }
订单 item 中可获取产品、数量、小计等信息。

2. 读取订单 item meta

基础 · php
<?php foreach ( $order->get_items() as $item ) { $engraving = $item->get_meta( 'Engraving' ); if ( $engraving ) { echo esc_html( $engraving ); } }
购物车自定义数据保存到订单 item 后，可在这里读取。

3. 给订单 item 添加内部 meta

实用 · php
<?php function mywoo_add_line_item_internal_meta( $item, $cart_item_key, $values, $order ) { if ( ! empty( $values['production_batch'] ) ) { $item->add_meta_data( '_production_batch', $values['production_batch'], true ); } } add_action( 'woocommerce_checkout_create_order_line_item', 'mywoo_add_line_item_internal_meta', 10, 4 );
以下划线开头的 meta 通常不在前台明显展示，适合内部信息。

4. 创建部分退款

进阶 · php
<?php $refund = wc_create_refund( array( 'amount' => 10.00, 'reason' => 'Partial refund for shipping adjustment.', 'order_id' => 123, 'refund_payment' => false, 'restock_items' => false, ) ); if ( is_wp_error( $refund ) ) { error_log( $refund->get_error_message() ); }
真实退款可能涉及支付网关 refund_payment，务必测试。

5. 保存物流单号到订单 meta

实用 · php
<?php $order = wc_get_order( 123 ); if ( $order ) { $order->update_meta_data( '_tracking_number', 'TRACK123456' ); $order->add_order_note( 'Tracking number added: TRACK123456' ); $order->save(); }
物流信息可保存到订单 meta，也可用专业物流插件。

6. 邮件中显示物流单号

实用 · php
<?php function mywoo_email_tracking_number( $order, $sent_to_admin, $plain_text, $email ) { $tracking = $order->get_meta( '_tracking_number' ); if ( $tracking ) { echo '<p><strong>Tracking Number:</strong> ' . esc_html( $tracking ) . '</p>'; } } add_action( 'woocommerce_email_after_order_table', 'mywoo_email_tracking_number', 10, 4 );
邮件输出要注意 HTML 和纯文本兼容。

本页总结

订单 item 和退款代码直接影响客户、库存和财务，必须谨慎测试，尤其是支付网关退款和库存回补。
