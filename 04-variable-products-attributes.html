代码使用提醒

本页代码用于学习和研究。复制到正式 WooCommerce 网站前，请先备份，并优先在测试环境验证。

凡是影响价格、库存、支付、订单、邮件和客户数据的代码，都需要完整测试购物车、结账、付款、退款、邮件和后台订单处理流程。

1. 获取产品属性

基础 · php
<?php $product = wc_get_product( 123 ); if ( $product ) { foreach ( $product->get_attributes() as $attribute ) { echo esc_html( wc_attribute_label( $attribute->get_name() ) ); } }
属性可能是全局 taxonomy，也可能是产品自定义属性。

2. 创建可变产品基础对象

进阶 · php
<?php $product = new WC_Product_Variable(); $product->set_name( 'Composite Decking Board' ); $product->set_status( 'publish' ); $product_id = $product->save();
可变产品本身不直接售卖，通常通过 variation 售卖。

3. 给产品设置自定义属性

进阶 · php
<?php $product = wc_get_product( $product_id ); $attribute = new WC_Product_Attribute(); $attribute->set_name( 'Color' ); $attribute->set_options( array( 'Teak', 'Walnut', 'Grey' ) ); $attribute->set_position( 0 ); $attribute->set_visible( true ); $attribute->set_variation( true ); $product->set_attributes( array( $attribute ) ); $product->save();
set_variation(true) 表示该属性可用于变体。

4. 创建变体

进阶 · php
<?php $variation = new WC_Product_Variation(); $variation->set_parent_id( $product_id ); $variation->set_attributes( array( 'Color' => 'Teak', ) ); $variation->set_regular_price( '59.99' ); $variation->set_manage_stock( true ); $variation->set_stock_quantity( 10 ); $variation->save();
变体属性 key/value 要和父产品属性对应。

5. 读取可变产品的可用变体

基础 · php
<?php $product = wc_get_product( 123 ); if ( $product && $product->is_type( 'variable' ) ) { $variations = $product->get_available_variations(); foreach ( $variations as $variation ) { echo esc_html( $variation['variation_id'] . ' - ' . $variation['display_price'] ); } }
get_available_variations 返回数组，适合前台展示。

6. 变体选择后追加自定义数据

实用 · php
<?php function mywoo_available_variation_extra_data( $data, $product, $variation ) { $data['custom_lead_time'] = get_post_meta( $variation->get_id(), '_lead_time', true ); return $data; } add_filter( 'woocommerce_available_variation', 'mywoo_available_variation_extra_data', 10, 3 );
可把变体额外数据传给前端 variation JS 使用。

本页总结

可变产品代码比简单产品复杂，核心是父产品属性、variation 对象、变体价格库存和前端变体数据。
